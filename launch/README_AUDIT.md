# README audit

Single most-quotable line:

`The agent owns the loop. You own the data. The graph compounds.` - README.md line 253.

Single most effective first-screen edit:

Before, README.md line 9:

`*Not a coding agent. Not a chatbot. An autonomous agent for the research journey - from "I have no idea what to work on" to your thesis defense.*`

After:

`*A local research agent for PhDs: maps fields, reads papers, checks citations, and compounds your library in ~/.peer.*`

Why this edit: the current line has personality, but it spends the first screen defining what peer is not. The replacement names the buyer, the surface, the core jobs, and the durable local memory in one sentence.

Single missing feature that would most help virality:

Add `peer share <topic>`: generate an anonymized, static share page from a field map or knowledge graph, with paper titles, subfields, reading order, citations, and the approximate run cost, but no private notes. The output should be one local HTML file plus an optional PNG preview.

Why it matters: peer's strongest output is visual and cumulative, but the current sharing path is implicit. A researcher needs a clean artifact they can post on Twitter/X, HN, Reddit, Slack, or a lab group chat without exposing their whole `~/.peer/` library. That makes the product legible in one screenshot.
