# Show HN: peer, a terminal research agent for PhDs

peer is a terminal-native research agent for PhDs and researchers. It is not a coding agent and not a chatbot UI. You run commands like `peer map "<topic>"`, `peer read <arxiv-id>`, `peer ask "<question>"`, `peer cite "<claim>"`, and `peer graph`. It builds a local `~/.peer/` workspace with SQLite, Markdown/Obsidian notes, an interactive D3 graph, and a role-aware profile for PhD students, postdocs, faculty, industry researchers, and independent learners. Install from source for now: `git clone https://github.com/KeWang0622/peer.git && cd peer && npm install && npm run build && npm link`, then run `peer doctor` and `peer onboard`. Because it is pre-1.0, there is no npm package yet; the README says npm publish under `@kewang/peer` is coming soon. You bring `ANTHROPIC_API_KEY` for the agent and `OPENAI_API_KEY` for embeddings; Semantic Scholar is optional for higher rate limits.

I built it because I was frustrated by how much research work happens between tools: a browser tab for arXiv, a notes app, a citation manager, a chat window, a half-remembered reading list, and no durable memory of why a paper mattered. The goal is not to replace judgment. It is to make the routine loops less lossy: map a field, read a paper into your graph, ask cited questions over your own library, find sparse intersections, draft related work groups, and track what happened. The README is honest about rough edges: this is `v0.0.1-alpha.10`, full-PDF parsing and Zotero/BibTeX import are still backlog items, and it ships from source while pre-1.0.

The technically interesting bit is the agent boundary. peer is built on `pi-agent-core`: Claude plans and selects from 17 research tools while memory lives in a 4-layer local graph: citations, semantic embeddings, your notes, and action history. The agent is sandboxed to its peer home directory (`~/.peer/` or `$PEER_HOME`) with read/write/edit/grep/find/ls only and no shell/bash tool. So if you are thinking "isn't this just ChatGPT", no: the loop includes planning, tool calls, state, and files. Yes, it is another AI tool, but the bet is that locality and accumulated research context matter. And if you are wondering "why a CLI", it is because many researchers already live in terminals, LaTeX, Git, and plain files. GitHub: https://github.com/KeWang0622/peer
